QUESTION TWO
28 marks

Control Structures and Decision Making

Part (a) - Conditional Statements (10 marks)

Write a program that determines student grade classification based on GPA:

  • GPA ≥ 3.7: "First Class Honors"
  • 3.0 ≤ GPA < 3.7: "Upper Second Class"
  • 2.0 ≤ GPA < 3.0: "Lower Second Class"
  • 1.0 ≤ GPA < 2.0: "Third Class"
  • GPA < 1.0: "Fail"

The program should:

  • Read GPA from user input
  • Validate that GPA is between 0.0 and 4.0
  • Print the appropriate classification
  • Handle invalid input with an error message

Part (b) - Loops and Iteration (10 marks)

Write a program to manage course enrollment:

  • Start with an empty list of courses
  • Use a while loop to repeatedly ask for course codes
  • Add each valid course code to the list
  • Continue until user enters "DONE"
  • Print the total number of courses enrolled
  • Print all course codes using a for loop

Sample interaction:

Enter course code (or DONE to finish): COMP1005 Enter course code (or DONE to finish): COMP2005 Enter course code (or DONE to finish): DONE You are enrolled in 2 courses: COMP1005 COMP2005

Part (c) - Nested Loops (8 marks)

Create a multiplication table generator:

  • Ask user for the maximum number (1-12)
  • Validate the input range
  • Generate a formatted multiplication table
  • Use nested loops to create the table

Expected output for input 3:

1 2 3 1 1 2 3 2 2 4 6 3 3 6 9
QUESTION THREE

Functions and Code Organization

Part (a) - Function Creation (12 marks)

Create functions for student GPA calculations:

  • calculate_gpa(grades) - Takes a list of letter grades, returns GPA
  • letter_to_points(grade) - Converts letter grade to grade points
  • validate_grades(grades) - Checks if all grades are valid (A-F)

Grade point conversion: A=4.0, B=3.0, C=2.0, D=1.0, F=0.0

Part (b) - Function Parameters (10 marks)

Write a function process_student_data(name, grades, courses) that:

  • Takes student name, list of grades, and list of courses
  • Validates that grades and courses lists have same length
  • Calculates GPA using the function from part (a)
  • Returns a dictionary with student information
  • Handles invalid data with appropriate error messages

Part (c) - Function Testing (8 marks)

Write test code for your functions:

  • Create test data with valid student information
  • Call each function with appropriate arguments
  • Print the results in a formatted way

26 marks

Error Handling and Input Validation

Part (a) - Exception Handling (14 marks)

Create a robust student data entry system:

  • Wrap user input in try-except blocks
  • Handle ValueError for invalid numeric input
  • Handle TypeError for invalid operations
  • Provide meaningful error messages
  • Allow user to retry after errors
  • Log all errors to a file for debugging

Specific requirements:

try: # Your code here except ValueError as e: print(f"Invalid input: {e}") except TypeError as e: print(f"Type error: {e}") except Exception as e: print(f"Unexpected error: {e}")

Part (b) - Input Validation (12 marks)

Create validation functions for student data:

  • validate_student_id(id) - Must be 6-8 digits
  • validate_email(email) - Must contain @ and .
  • validate_gpa(gpa) - Must be between 0.0 and 4.0
  • validate_year(year) - Must be reasonable enrollment year

Each function should:

  • Return True if valid, False if invalid
  • Include appropriate validation logic
  • Handle edge cases appropriately
